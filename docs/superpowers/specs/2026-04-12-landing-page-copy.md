## ✅ SOLUTION / FEATURE SECTION

**Section label:** Built for the way you actually work

**Feature 1 — Parallel agents, no chaos**  
Open Claude Code on auth, Gemini CLI on the API, and Cursor on the UI — all at the same time. Each agent works on its own branch. No conflicts, no shared state.

**Feature 2 — Each agent, its own worktree**  
Every session gets an isolated git worktree. Changes stay separate until you're ready to review and merge. You decide what lands — not whoever wrote last.

**Feature 3 — Bring your own subscription**  
workbench doesn't charge on top of your API spend. Use the subscriptions you already have and pay only for what the models actually cost.

**Feature 4 — Review before you merge**  
A built-in diff viewer shows exactly what each agent changed. Compare across agents, pick the best result, and merge — without leaving the app.

## ❓ FAQ

**Q: Do I need to pay for workbench on top of my AI subscriptions?**  
A: No. workbench is free to download. You use the API keys or subscriptions you already have — there's no additional subscription on top.

**Q: Which agents does workbench support?**  
A: Claude Code, Gemini CLI, Cursor Agent, and OpenAI Codex work out of the box. Anything that runs in a terminal will work.

**Q: What's a git worktree and why does it matter?**  
A: A worktree is a separate checkout of your repo on a different branch. Each agent gets its own, so they can modify files independently without overwriting each other's work.

**Q: Does this work on Windows and Linux?**  
A: Yes — macOS, Windows, and Linux are all supported.

**Q: Is my code sent anywhere?**  
A: workbench doesn't process your code — it's just the shell around your agents. What gets sent to the cloud is between you and whatever agent you're running.
